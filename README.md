## 是什么

四元蒸馏框架是一个提示词优化系统，融合了四位顶级提示词工程师的核心方法论，将其提炼为四步可执行的工作流。

**核心价值**：把用户模糊的提示词，变成能激发AI高质量回答的精准指令。

## 快速开始

### 激活方式

直接粘贴提示词，或说以下任一触发词：

`
"帮我优化提示词"
"改进我的Prompt"
"提示词蒸馏"
"prompt alchemist"
"帮我看看这个提示词"
`

### 30 秒判断任务复杂度

| 判断依据 | 简单 | 中等 | 复杂 |
|---------|------|------|------|
| 原始字数 | < 100字 | 100-500字 | > 500字 |
| 任务类型 | 单一明确 | 多步骤/多维度 | 推理/创意/第三象限 |
| 结构缺失 | 1个小问题 | 缺2-3个元素 | 缺框架/角色/背景 |

## 核心机制：任务路由（强制入口）

**在开始蒸馏前，先判断任务类型。不同类型触发不同的约束模块和输出格式。**

| 任务类型 | 识别关键词 | 必须触发的特殊约束 |
|---------|-----------|----------------|
| **规划/路线/计划类** | 学习路线、学习计划、开发计划、产品规划、Roadmap | 最短路径+漏斗+可验收三原则 + 瓶颈预案表 |
| **汇报/述职类** | 报告、演示、述职、汇报、PPT | 结论先行 + 去技术化 + 问题不藏最后 |
| **问答/解释类** | 解释、说明、回答、定义、什么是 | 直接给结论 + 背景补充 |
| **创意/写作类** | 写、设计、创作、策划 | 角色光谱定位 + Few-Shot锚定 |
| **代码/技术类** | 代码、开发、程序、脚本、调试 | 技术栈约束 + 边界条件 + 输出格式 |

---

## 四步工作流

`
用户提供提示词
     ↓
  ⓪ 路由   → 判断任务类型，触发对应约束模块
  ① 诊断   → 识别问题类型和结构缺陷
  ② 结构   → 搭建清晰骨架（XML标签）
  ③ 思考   → 嵌入适当的思考深度
  ④ 压缩   → 蒸馏出精准成品
     ↓
输出优化后的提示词
`

---

## 规划类任务的强制约束

当任务类型为\"规划/路线/计划\"时，必须包含三条原则：

| 原则 | 含义 |
|------|------|
| **最短路径原则** | 路线不是知识清单，是从起点到终点的最短路径 |
| **漏斗原则** | 每阶段内容≤5项，超过必须拆成两段 |
| **可验收原则** | 每阶段必须有具体产出物（能独立完成___） |

> ⚠️ 瓶颈预案表必须写，不写=任务完成度不足
