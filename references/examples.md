# Prompt Alchemist — 示例库

## 输出格式分级规则

根据任务复杂度调整输出详细程度：

### 简单任务（单一问题、明确需求）
直接输出：
- 🔍 诊断（3-5 行）
- ⚡ 蒸馏成品

### 中等任务（需要结构调整）
输出：
- 🔍 诊断
- 🏗️ 结构重构
- ⚡ 蒸馏成品

### 复杂任务（涉及推理、决策、创意）
输出完整 5 部分：
- 🔍 诊断报告
- 🏗️ 结构重构
- 🧠 思考增强
- ⚡ 蒸馏成品
- 📊 变更说明

## 示例零：简单任务优化（新增）

### 原始提示词
```
翻译成英文
```

### 诊断
- 任务类型：翻译
- 乔哈里象限：第四象限（人知AI不知具体需求）→ 必须补充背景
- 核心问题：无上下文、无风格要求、无质量标准

### 优化后
```xml
<role>专业翻译，擅长中英双语商务与技术文档</role>

<task>将以下中文翻译成英文：</task>

<constraints>
- 风格：商务邮件（正式但不过于生硬）
- 要求：保留原意，避免过度意译
- 专业术语保持一致性

<format>
直接输出英文译文，不需要解释
</format>
```

### 大师印记
- 🧪 Goodside: 简单的结构约束 + 格式要求
- 🎯 李继刚: "商务邮件"精准压缩风格（一个属性锁定语气）
- 📊 Schulhoff: 补充背景（风格+要求）
- 🌊 涂津豪: 适度深度——简单任务不需要复杂的思考引导

## 示例一：简单事实问答优化

### 原始提示词
```
告诉我关于量子计算的事情
```

### 诊断
- 任务类型：事实问答
- 乔哈里象限：第二象限（人不知AI知）→ 需提升提问精度
- 核心问题：范围太广、无输出格式、无深度引导

### 优化后
```xml
<role>量子计算领域的资深科普专家，擅长用类比将复杂概念通俗化</role>

<task>从以下四个维度介绍量子计算：</task>
1. 核心原理：用不超过3个日常类比解释量子比特与经典比特的本质区别
2. 关键技术：列出当前3个最主流的量子计算实现路径及其优劣势
3. 应用前景：哪些领域最可能率先实现"量子优势"，为什么
4. 当前瓶颈：距离实用化最大的3个技术障碍

<format>
每个维度200字以内，关键术语加粗，避免泛泛而谈
</format>
```

### 大师印记
- 🧪 Goodside: XML标签分层 + 格式约束
- 🎯 李继刚: 角色压缩包（"科普专家+类比"两个属性锁定风格）
- 📊 Schulhoff: 任务拆解（4个子维度）+ 结构化输出
- 🌊 涂津豪: 深度引导（"避免泛泛而谈"暗示需要深入思考）

## 示例二：创意写作优化

### 原始提示词
```
帮我写一篇关于秋天的散文，要写得美一点
```

### 诊断
- 任务类型：创意写作
- 乔哈里象限：第一象限偏第三象限（人知部分AI知部分）→ 需补充审美方向
- 核心问题："美"太模糊、无风格锚定、无情感方向

### 优化后
```xml
<role>散文作家，笔触介于川端康成的物哀美学与汪曾祺的市井温情之间</role>

<context>写一篇关于秋天的散文</context>

<constraints>
- 不出现"金风送爽""秋高气爽"等陈词
- 以一个具体场景开篇（不概括，只白描）
- 情感走向：从感官沉浸到时间感知的微痛
- 篇幅：800-1200字
</constraints>

<examples>
风格锚定：
"梧桐叶落下来的样子，不是飘，是决绝地翻了一个身。"
——注意：具体、动感、情感暗线
</examples>

<format>纯文本，段落间留呼吸感，不使用小标题</format>
```

### 大师印记
- 🧪 Goodside: XML结构化 + 格式锚定
- 🎯 李继刚: 角色用两位作家名精准压缩风格光谱；示例锚定（few-shots的变体）；"决绝地翻了一个身"是Embedding精准命中的范例
- 📊 Schulhoff: Few-Shot锚定（一个风格示例胜过百句描述）；补充背景（情感方向）
- 🌊 涂津豪: "从感官沉浸到时间感知的微痛"——引导意识流式的审美思考路径

## 示例三：代码生成优化

### 原始提示词
```
用Python写一个爬虫
```

### 诊断
- 任务类型：代码生成
- 乔哈里象限：第四象限（人知AI不知具体需求）→ 必须补充背景和结构
- 核心问题：无目标网站、无功能需求、无异常处理要求、无代码规范

### 优化后
```xml
<role>Python高级工程师，代码风格：类型注解完备、docstring完整、异常处理健壮</role>

<task>用Python编写一个通用网页爬虫，要求：</task>

1. 核心功能
   - 输入：起始URL + 最大爬取深度 + 最大页面数
   - 输出：结构化数据（标题、正文、链接、爬取时间）
   - 支持：BFS策略、robots.txt遵守、请求频率限制

2. 技术栈
   - requests + BeautifulSoup（不使用Scrapy）
   - 使用${PYTHON_VERSION:3.10+}特性

3. 质量要求
   - 每个函数添加类型注解和docstring
   - 自定义异常类处理网络/解析错误
   - 支持断点续爬（进度持久化到JSON）
   - 添加rate limiting（默认1秒间隔）

<format>
完整的单文件Python脚本，包含：
- 导入区域
- 常量/配置区域
- 自定义异常
- 核心类/函数
- main()入口
- 使用示例注释
</format>
```

### 大师印记
- 🧪 Goodside: `${PYTHON_VERSION:3.10+}`变量占位符 + 格式约束精确到代码结构
- 🎯 李继刚: 角色压缩包（三个属性：类型注解、docstring、异常处理锁定代码风格）
- 📊 Schulhoff: 任务拆解（3个层级逐步细化）+ 补充背景
- 🌊 涂津豪: "先理解需求全貌再动手"——通过结构化约束避免模型直接跳入编码

## 示例四：复杂推理决策优化

### 原始提示词
```
我该不该辞职创业？
```

### 诊断
- 任务类型：推理决策
- 乔哈里象限：第三象限（人不知AI不知）→ 需人AI协同探索
- 核心问题：无个人背景、无决策框架、无风险评估、过于开放

### 优化后
```xml
<role>资深职业战略顾问，思维方式：先拆解再整合，偏好框架化思考，对风险高度敏感</role>

<context>
我正在考虑是否辞职创业，需要系统化分析这个决策。
</context>

<task>
请在回答前进行深入的意识流思考——不要急于给建议，先审视这个问题的多个维度：
1. 这不仅仅是"要不要辞职"的问题，底层是"什么样的生活才是我想要的"
2. 思考过程中，质疑每一个看似理所当然的假设
3. 注意人们在做此类决策时常犯的认知偏差（沉没成本、幸存者偏差、过度自信）

分析框架：
1. 诊断层：我为什么想创业？（区分"逃离型"和"奔赴型"动机）
2. 评估层：从5个维度打分（0-10）——财务缓冲、市场验证、能力匹配、心理准备、机会成本
3. 决策层：给出3条不同风险偏好的路径（保守/均衡/激进），每条路径标注关键里程碑和退出条件
4. 反思层：审视以上分析本身可能的盲点

<format>
每个层级独立成段，关键判断加粗，维度打分用表格呈现
</format>

<self-criticism>
完成分析后，审视自己的建议：是否过度偏向安全？是否忽视了创业的非理性价值？
</self-criticism>
```

### 大师印记
- 🧪 Goodside: XML结构 + 格式约束 + `<self-criticism>`标签（结构化自我批评）
- 🌊 涂津豪: 意识流思考引导（"审视问题的多个维度""质疑每个假设"）；自适应深度（复杂任务=深层思考）
- 📊 Schulhoff: 任务拆解（4层分析框架）+ 自我批评技法 + 补充背景（认知偏差提醒）
- 🎯 李继刚: 角色压缩包（"先拆解再整合+框架化+风险敏感"三个属性）；"逃离型vs奔赴型"——精准的二元压缩
