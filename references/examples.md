# Prompt Alchemist — 示例库

## 输出格式分级规则

| 复杂度 | 判断标准 | 输出模块 |
|--------|----------|----------|
| 简单 | 单一问题、明确需求、< 100字原始提示词 | 🔍 诊断（3-5行）+ ⚡ 蒸馏成品 |
| 中等 | 需要结构调整、多步骤任务 | 🔍 诊断 + 🏗️ 结构重构 + ⚡ 蒸馏成品 |
| 复杂 | 涉及推理/决策/创意、乔哈里第三象限 | 🔍 诊断报告 + 🏗️ 结构重构 + 🧠 思考增强 + ⚡ 蒸馏成品 + 📊 变更说明 |

> ⚠️ 不要对简单任务过度设计。如果诊断结果只有1个小问题，只输出诊断+成品并注明"微小优化"。

## 示例一：简单翻译任务

### 原始提示词
```
翻译成英文
```

### 🔍 诊断
- **原始质量**：⭐⭐ (2/5) — 缺少翻译对象和风格约束
- **任务类型**：翻译
- **乔哈里象限**：第四象限（人知AI不知）→ 必须补充背景
- **核心问题**：无翻译内容、无风格要求、无质量标准

### ⚡ 蒸馏成品
```xml
<role>专业翻译，擅长中英双语商务与技术文档</role>

<task>将以下中文翻译成英文：</task>

[在此粘贴待翻译的中文内容]

<constraints>
- 风格：商务邮件（正式但不过于生硬）
- 保留原意，避免过度意译
- 专业术语保持一致性
</constraints>

<format>直接输出英文译文，不需要解释</format>
```

**大师印记**：
- 🧪 Goodside: 最小结构约束 + 格式要求
- 🎯 李继刚: "商务邮件"精准压缩风格
- 📊 Schulhoff: 补充背景（风格+要求）
- 🌊 涂津豪: 简单任务不需要复杂思考引导

## 示例二：事实问答优化

### 原始提示词
```
告诉我关于量子计算的事情
```

### 🔍 诊断
- **原始质量**：⭐⭐ (2/5) — 范围太广无聚焦
- **任务类型**：事实问答
- **乔哈里象限**：第二象限（人不知AI知）→ 需提升提问精度
- **核心问题**：范围太广、无输出格式、无深度引导

### ⚡ 蒸馏成品
```xml
<role>量子计算领域的资深科普专家，擅长用类比将复杂概念通俗化</role>

<task>从以下四个维度介绍量子计算：</task>

1. 核心原理：用不超过3个日常类比解释量子比特与经典比特的本质区别
2. 关键技术：列出当前3个最主流的量子计算实现路径及其优劣势
3. 应用前景：哪些领域最可能率先实现"量子优势"，为什么
4. 当前瓶颈：距离实用化最大的3个技术障碍

<constraints>
- 每个维度200字以内
- 关键术语加粗
- 避免泛泛而谈
</constraints>

<format>
分点输出，每点一个小标题，关键术语用**加粗**
</format>
```

**大师印记**：
- 🧪 Goodside: XML标签分层 + 格式约束
- 🎯 李继刚: 角色压缩包（"科普专家+类比"锁定风格）
- 📊 Schulhoff: 任务拆解（4个子维度）+ 结构化输出
- 🌊 涂津豪: 深度引导（"避免泛泛而谈"暗示深入）

## 示例三：创意写作优化

### 原始提示词
```
帮我写一篇关于秋天的散文，要写得美一点
```

### 🔍 诊断报告
- **原始质量**：⭐⭐ (2/5) — "美"太模糊无锚定
- **任务类型**：创意写作
- **乔哈里象限**：第一象限偏第三象限 → 需补充审美方向
- **核心问题**："美"太主观、无风格锚定、无情感方向
- **优化方向**：用具体作家风格光谱 + Few-Shot示例锚定审美

### 🏗️ 结构重构
- **角色定义**：散文作家，风格介于川端康成与汪曾祺之间
- **任务拆解**：开篇白描 → 感官沉浸 → 时间感知
- **输出格式**：纯文本，段落留呼吸感

### 🧠 思考增强
- **深度级别**：中等（创意任务需要审美判断）
- **思考引导**：不直接写"悲伤"，用白描让读者自己感受

### ⚡ 蒸馏成品
```xml
<role>散文作家，笔触介于川端康成的物哀美学与汪曾祺的市井温情之间</role>

<task>写一篇关于秋天的散文</task>

<constraints>
- 不出现"金风送爽""秋高气爽""层林尽染"等陈词
- 以一个具体场景开篇（不概括，只白描）
- 情感走向：从感官沉浸到时间感知的微痛
- 篇幅：800-1200字
</constraints>

<examples>
风格锚定：
"她坐在那里剥豆子，手很慢，豆子从指缝间漏下去的声音好像房子里的另一个人的呼吸。"
——注意：用身体动作代替情感描写，不写"她感到孤独"，让读者自己感受
</examples>

<format>纯文本，段落间留呼吸感，不使用小标题</format>
```

### 📊 变更说明

| 维度 | 原始 | 优化后 | 改进依据 |
|------|------|--------|----------|
| 角色 | 无 | 散文作家（川端康成-汪曾祺光谱） | 李继刚：角色压缩包精准定位风格 |
| 风格 | "美一点" | 物哀美学+市井温情 | Goodside：用标签显式化风格 |
| 约束 | 无 | 禁用陈词+白描开篇+情感走向 | Schulhoff：补充背景+负面约束 |
| 示例 | 无 | 一个白描风格的句子 | Schulhoff：Few-Shot锚定效果强于描述 |

**大师印记**：
- 🧪 Goodside: XML结构化 + 格式锚定
- 🎯 李继刚: 用两位作家名精准压缩风格光谱
- 📊 Schulhoff: Few-Shot锚定（一个风格示例胜过百句描述）
- 🌊 涂津豪: "从感官沉浸到时间感知的微痛"——引导意识流式审美路径

## 示例四：代码生成优化

### 原始提示词
```
用Python写一个爬虫
```

### 🔍 诊断报告
- **原始质量**：⭐ (1/5) — 几乎无法执行
- **任务类型**：代码生成
- **乔哈里象限**：第四象限（人知AI不知）→ 必须补充完整背景
- **核心问题**：无目标网站、无功能需求、无异常处理要求、无代码规范
- **优化方向**：结构化需求模板 + 技术栈约束 + 质量要求

### 🏗️ 结构重构
- **角色定义**：Python高级工程师，代码风格明确
- **任务拆解**：核心功能 / 技术栈 / 质量要求
- **输出格式**：完整单文件脚本结构

### 🧠 思考增强
- **深度级别**：中等（代码生成需要系统性思考）
- **思考引导**：先理解需求全貌再动手

### ⚡ 蒸馏成品
```xml
<role>Python高级工程师，代码风格：类型注解完备、docstring完整、异常处理健壮</role>

<task>用Python编写一个通用网页爬虫</task>

<constraints>
核心功能：
- 输入：起始URL + 最大爬取深度 + 最大页面数
- 输出：结构化数据（标题、正文、链接、爬取时间）
- 支持：BFS策略、robots.txt遵守、请求频率限制

技术栈：
- 使用 requests + BeautifulSoup（不使用Scrapy）
- 兼容 Python 3.10+ 特性

质量要求：
- 每个函数添加类型注解和docstring
- 自定义异常类处理网络/解析错误
- 支持断点续爬（进度持久化到JSON）
- 添加rate limiting（默认1秒间隔）
</constraints>

<format>
完整的单文件Python脚本，包含：
- 导入区域
- 常量/配置区域  
- 自定义异常
- 核心类/函数
- main()入口
- 使用示例注释
</format>
```

### 📊 变更说明

| 维度 | 原始 | 优化后 | 改进依据 |
|------|------|--------|----------|
| 角色 | 无 | Python高级工程师（三属性锁定风格） | 李继刚：角色压缩包 |
| 结构 | 一句话 | 三层级需求模板 | Schulhoff：任务拆解 |
| 约束 | 无 | 技术栈+功能+质量 | Goodside：结构化约束 |
| 格式 | 无 | 完整脚本结构 | Goodside：格式锚定 |

**大师印记**：
- 🧪 Goodside: 变量占位思想 + 格式约束精确到代码结构
- 🎯 李继刚: 角色压缩包（类型注解+docstring+异常处理锁定代码风格）
- 📊 Schulhoff: 任务拆解（3层级逐步细化）+ 补充背景
- 🌊 涂津豪: "先理解需求全貌再动手"——通过结构化约束避免跳过思考

## 示例五：复杂推理决策优化

### 原始提示词
```
我该不该辞职创业？
```

### 🔍 诊断报告
- **原始质量**：⭐⭐ (2/5) — 过于开放无框架
- **任务类型**：推理决策
- **乔哈里象限**：第三象限（人不知AI不知）→ 需人AI协同探索
- **核心问题**：无个人背景、无决策框架、无风险评估、过于开放
- **优化方向**：四层分析框架 + 自我批评机制 + 意识流思考引导

### 🏗️ 结构重构
- **角色定义**：职业战略顾问，框架化思维，风险敏感
- **任务拆解**：诊断层 → 评估层 → 决策层 → 反思层
- **输出格式**：分段+表格+加粗关键判断

### 🧠 思考增强
- **深度级别**：复杂（涉及人生重大决策）
- **思考引导**：意识流思考——审视问题本质、质疑假设、回溯逻辑链
- **自我批评**：审视建议是否过度偏向安全或忽视非理性价值

### ⚡ 蒸馏成品
```xml
<role>资深职业战略顾问，思维方式：先拆解再整合，偏好框架化思考，对风险高度敏感</role>

<context>我正在考虑是否辞职创业，需要系统化分析这个决策。</context>

<task>请按以下框架进行分析：</task>

诊断层：我为什么想创业？
- 区分"逃离型"动机（想逃离当前困境）和"奔赴型"动机（有明确目标）
- 审视动机背后的真实需求

评估层：从5个维度打分（0-10）
- 财务缓冲：现有资金能支撑多久？
- 市场验证：想法是否有初步验证？
- 能力匹配：核心能力是否覆盖？
- 心理准备：能否承受失败？
- 机会成本：放弃的稳定收益有多大？

决策层：给出3条路径
- 保守路径：边工作边验证，里程碑是____
- 均衡路径：预留6个月缓冲，里程碑是____
- 激进路径：全职投入，里程碑是____
- 每条路径标注关键退出条件

反思层：审视以上分析
- 是否过度偏向安全？
- 是否忽视了创业的非理性价值（成长、意义感）？

<format>
每个层级独立成段，关键判断加粗，维度打分用表格呈现
</format>

<self-criticism>
完成分析后，回答：我的建议是否过于理性？是否遗漏了你个人情况中的关键因素？
</self-criticism>
```

### 📊 变更说明

| 维度 | 原始 | 优化后 | 改进依据 |
|------|------|--------|----------|
| 角色 | 无 | 职业战略顾问（三属性） | 李继刚：角色压缩包 |
| 结构 | 一句话 | 四层分析框架 | Schulhoff：任务拆解 |
| 思考 | 无 | 意识流思考引导 | 涂津豪：自适应深度 |
| 自检 | 无 | 自我批评标签 | Schulhoff：自我批评技法 |

**大师印记**：
- 🧪 Goodside: XML结构 + `<self-criticism>` 标签（结构化自我批评）
- 🌊 涂津豪: 意识流思考引导（"审视问题的本质、质疑每个假设"）
- 📊 Schulhoff: 任务拆解（4层框架）+ 自我批评技法
- 🎯 李继刚: "逃离型vs奔赴型"——精准的二元压缩

## 反模式对照表

| 反模式 | 错误示例 | 正确做法 |
|--------|----------|----------|
| 过度设计简单任务 | 翻译"Hello"输出五段式 | 简单任务只输出诊断+成品 |
| 把方法论写进提示词 | `<task>请应用涂津豪的意识流思考</task>` | 不暴露框架，直接嵌入思考引导 |
| 角色风格冲突 | `<role>脱口秀演员</role>` + `<tone>正式严谨</tone>` | 检查三者一致性 |
| 忘记约束保全 | 原文"诚恳+不显得理亏"，压缩后只剩"道歉邮件" | 执行步骤⑥逐条对照 |

## 自检清单

蒸馏完成后，逐项确认：

- [ ] 步骤⑥"约束保全"是否执行？所有原始约束都有落点？
- [ ] 步骤⑦"一行测试"是否通过？能否一句话概括核心？
- [ ] 步骤⑧"共振检验"是否通过？AI回答是否有本质变化？
- [ ] 是否有反模式？特别是"把方法论写进提示词"？
- [ ] 复杂度判断是否正确？简单任务是否过度输出？
- [ ] 变更说明是否只列出实际变更的维度？
