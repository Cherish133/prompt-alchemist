> 透明化本框架的边界：什么情况下会失效、什么场景适合/不适合、已知的模型差异。
> **版本**：v1.3.0 | 最后更新：2026-06-26

---

## 一、核心失效模式

### FM-1：过量优化（Over-Engineering）

**症状**：蒸馏后的提示词**比原始版更差**或**更长更冗余**。

**触发条件**：
- 原始提示词已经很好了（尤其是付费用户的"测试版"）
- 简单问题（翻译、单步查询）用完整四步框架
- 用户只需要"稍微调整"，不需要全面改写

**实测案例（TP-05）**：
- 原始："翻译成英文"
- v1.0.1 蒸馏输出：4段 XML 模板（实际上需要的是追问"翻译什么文本？"）
- 教训：遇上 no-op 任务（无可优化空间时），输出模板不如直接追问

**判断标准**：
```
蒸馏后字数 / 原始字数 > 2.5 → 触发二次压缩检查
蒸馏后字数 > 1000 且是简单任务 → 触发回退策略
```

**修正**：回退策略三次蒸馏（只留 `<role>+<task>+<format>`）

---

### FM-2：乔哈里象限误判

**症状**：对任务所在象限的判断错误，导致整体策略错误。

**触发条件**：
- 用户隐晦表达（"你懂的"）→ AI 误判为第一象限，实际是第二/三象限
- 专业领域（用户以为 AI 懂，但 AI 只是表面懂）
- 用户提供的信息不足（以为用户已知，实际缺失）

**实测案例（TP-04）**：
- 原始："我该不该辞职创业？"（典型第二象限）
- v1.0.1 正确识别为第二象限
- 如果误判为第一象限 → 直接输出"创业建议"，效果极差

**判断标准**：
```
用户信息 < 3 条 → 默认是第二/三/四象限
用户提供明确数据 → 第一象限
用户说"你觉得" / "你怎么看" → 第二象限
```

**修正**：不确定时标注"基于现有信息，默认按第 X 象限处理，待用户补充信息后重新定位"

---

### FM-3：中文思维链退化

**症状**：思维链技术在中文提示词中**效果明显低于英文**。

**触发条件**：
- 用直译的"逐步思考"替代"Let's think step by step"
- 中文复杂的修辞结构破坏思维链的线性逻辑
- 模型对中文 CoT 的训练不足

**实证**：Schulhoff 1565篇论文只有少数涉及中文，五大技法的效果评级基于英文

**修正**：
- 中文语境偏好"展示思路"而非"告诉我们步骤"
- 用意识流（涂津豪风格）替代结构化 CoT
- 长推理用"先思考 X，再决定 Y"代替"Step 1. Step 2."

> ✅ **v1.7.0 已修复**：SKILL.md 第三步"思考增强"中已新增中文 CoT 退化警告，明确"中文优先用任务拆解，慎用直译 CoT"，并给出中文意识流替代句式示例。

---

### FM-4：模型差异导致的"迁移失败"

**症状**：同一提示词在不同模型上效果差异大，但框架未做适配。

**触发条件**：
- 未指定目标模型时采用通用策略
- 用户认为"写一个提示词就够了"（实际上不通）
- 框架的 XML 标签在 GPT 上工作良好，Gemini 可能忽略

**修正**：
- 标注 `${TARGET_MODEL:claude}` 占位（即使未指定）
- 见 `references/model-adaptation.md` 的三模型定制策略

**退出选项**：当 P0 与 P2 约束物理冲突且无法降级时，在蒸馏成品中显式提示用户选择：
> "以下两个约束在物理上不可同时满足，请选择：
> A. [保留核心功能，放宽字数/格式限制]
> B. [强制字数限制，将部分功能降级为说明而非实现]"
- 若用户未选择，默认执行选项 A（功能优先，字数为参考约束）

---

### FM-5：Few-Shot 示例顺序反向优化

**症状**：加了示例后，效果比没有示例更差。

**触发条件**：
- 示例与任务类型不匹配（Schulhoff 的"Few-Shot 错位"）
- 示例顺序使模型学会错误模式
- 示例数量 > 5 个，反而稀释效果

**实证**：
- Schulhoff 研究：仅调整示例顺序，准确率可从 50% 波动到 90%
- TP-02 实测：
  - 差示例放在前面 → 模型学会差的风格
  - 好示例放在前面 → 模型自动对齐

**修正**：
- 最多 2-3 个正反对比示例
- 正例在前，反例在后
- 示例必须与当前任务同类型

---

### FM-6：知识截止冲突

**症状**：用户询问实时/未来/最新数据（政策/法规/价格/新闻），但模型知识存在截止日期，框架无显式声明规则。

**触发条件**：
- 用户输入含"最新""2025年""当前""现在"等时效词
- 涉及政策、法规、股价、新闻、天气等知识库不稳定内容
- 用户问"现在/今年/最新"相关问题

**实证**：模型知识库有明确截止日期（如 GPT-4o 截至 2023-12，Claude 截至 2024-04，DeepSeek 截至 2024-06），用户询问 2025 年数据时，框架若不声明知识边界，会导致：① 幻觉填充未来数据 ② 用户基于虚假信息做重大决策 ③ 信任崩塌

**修正**：蒸馏成品必须包含知识边界声明，嵌入 `<context>` 标签：
- **截止声明**：`本回答基于模型知识库截至 [知识库日期] 的信息撰写`
- **核实渠道**：给出 1-3 个官方查询渠道（政府官网/权威媒体/数据库链接）
- **结论措辞**：用"预计/可能/大概率"替代确定性表述；禁止写"一定会涨/跌/实施"
- **用户决策**：明确告知"如有重要决策需求，请以官方最新公告为准"

**框架要求**：不声明知识截止的蒸馏成品视为框架遗漏。

## 二、已知局限

### 2.1 学术基础不足

- **英文优先**：Schulhoff 的 1565 篇论文集中在英文，中文效果可能不同
- **论文截止 2023**：AI 领域发展快，部分技法可能已被模型进化吸收
- **Goodside 的 Format Trick**：对 Claude 最有效（XML 标签友好），对 Gemini 效果存疑

### 2.2 框架结构局限

- **复杂度过高**：四步日常用可能太长，需要快速通道
- **中等任务分级模糊**：输出详细度自适应是文本规则，不是自动分级工具
- **回退策略依赖人工判断**：什么是"更长/更模糊"的标准存在主观性

### 2.3 模型生态局限

- **ChatGPT 作为反馈弱**：本框架的评估依赖人工，不能通过 ChatGPT 验证效果
- **上下文窗口差异大**：200K vs 8K 的适配规则缺失

### 2.4 用户使用局限

- **新手不理解四步框架**：输出"大师印记"可能让非专业用户困惑
- **中文社区传播度有限**：四位大师的中文资料分布零散
- **无批量处理**：一次只能处理一个提示词

### 2.5 版本兼容

- **v1.3.0 实测 4.98/5**：接近上限，后续优化边际收益递减
- **快速参考卡可能过时**：随着模型升级，Top 5 修复权重会变化

## 三、场景适用性矩阵

| 场景 | 推荐使用 | 可能失效 | 原因 |
|------|---------|---------|------|
| 简单的单步翻译 | ❌ | ✅ FM-1 过量优化 | 框架太重 |
| "帮我写一个"（纯命令） | ✅ | ❌ | 预检能识别并追问 |
| 复杂编码任务 | ✅ | ❌ | 结构约束强 |
| 创意写作/散文 | ✅ | ✅ FM-5 Few-Shot错位 | 示例设计需谨慎 |
| 人生决策/咨询 | ✅ | ✅ FM-2 象限误判 | 依赖正确象限判断 |
| 多轮对话提示词 | ❌ | ✅ | 框架针对单轮设计 |
| 嵌入在代码中的prompt | ❌ | ✅ | 不适合 XML 结构 |

## 五、快速诊断：我的提示词出了什么问题？

```
提示词有问题吗？
│
├─ 蒸馏后比原始更长 → FM-1 过量优化 → 回退策略
├─ 方向跑偏（AI回复不是你要的） → FM-2 象限误判 → 补充背景/重新定位
├─ 中文语境效果差 → FM-3 中文CoT退化 → 改用意识流 ✅ v1.7.0已修复
├─ 换模型后不work → FM-4 模型不兼容 → 看 model-adaptation.md
├─ 加了示例更差 → FM-5 示例错位 → 重新设计示例顺序
│
├─ 多轮对话后角色/约束漂移 → AP-M1/AP-M2 → 查M-4状态同步
├─ 多轮后提示词越来越长 → AP-M3 上下文爆炸 → 每轮强制检查<role> ✅ v1.7.0已修复
│
├─ 问最新政策/实时数据 → FM-6 知识截止冲突 → 加<context>截止声明+核实渠道 ✅ v1.8.0新增
│
└─ 都不是 → 提交 issue 到 @alchaincyf
```
