## 一、核心测试集（5 题）

### TP-01：事实问答（简单）
```text
输入：请解释量子计算的基本原理
难度：简单
目标：评估基本结构化能力
反模式风险：AP-8（元数据缺失）、AP-3（模糊形容词"基本原理"）
```

### TP-02：创意写作（中等）
```text
输入：帮我写一篇关于秋天的散文，要美一点
难度：中等
目标：评估模糊表述精炼 + 风格锚定能力
反模式风险：AP-3（"美一点"典型）、AP-4（任务混淆"散文+美"）
失效模式风险：FM-5（示例错位）
```

### TP-03：代码生成（中等）
```text
输入：帮我写一个Python爬虫，爬取网页内容
难度：中等
目标：评估结构约束 + 模型适配能力
反模式风险：AP-6（元数据缺失）、AP-4（任务混淆）
失效模式风险：FM-4（模型适配）
```

### TP-04：推理决策（复杂）
```text
输入：我该不该辞职创业？
难度：复杂
目标：评估思考深度 + 乔哈里象限定位能力
反模式风险：AP-5（任务混淆）
失效模式风险：FM-2（象限误判）
```

### TP-05：翻译（极简 no-op）
```text
输入：翻译成英文
难度：简单（no-op）
目标：评估预检 + 回退策略
反模式风险：AP-8
失效模式风险：FM-1（过量优化）
```

## 三、评分细则

### 5 分制评分卡

| 分数 | 描述 | 判定标准 |
|------|------|---------|
| 5 | 完美 | 输出可直接使用，无需任何修改 |
| 4 | 优秀 | 结构完整，有一处小瑕疵（缺少可选元数据） |
| 3 | 合格 | 框架正确执行，但输出不够精准或过度优化 |
| 2 | 较差 | 框架部分执行，但关键环节出错或遗漏 |
| 1 | 不合格 | 未运行框架或方向性错误 |

### TRACES 评分维度

| 维度 | 权重 | 考核点 |
|------|------|--------|
| T · Trust | 15% | 安全与可信度，敏感内容处理 |
| R · Reliability | 20% | 异常输入处理（预检+回退），边界条件 |
| A · Adaptability | 20% | 模型适配、任务复杂度分级、服务边界 |
| C · Convention | 20% | 反模式避坑、元数据规范、大师印记 |
| E · Effectiveness | 25% | 蒸馏成品质量、结构清晰度、输出可用性 |

### 评分公式

```
总分 = T×0.15 + R×0.20 + A×0.20 + C×0.20 + E×0.25
```

## 四、测试方法

### 方法 A：独立 subagent 对比（推荐）

```
步骤：
1. 从 git checkout 旧版本 SKILL.md
2. 创建 subagent，prompt="你是 prompt-alchemist <版本> 的模拟器。加载 <SKILL.md> 并对 TP-XX 执行蒸馏"
3. 保存结果到 test-results-<版本>-<TP-XX>.md
4. 切换到新版本 SKILL.md，重复步骤2-3
5. 主会话对比评分
```

**实测时间**（5 prompts，单模型）：~2 分钟

### 方法 B：主会话直接跑（快速验证）

```
适用于单 TP 或少数 TP 验证。
直接将 SKILL.md 中核心部分嵌入提示词，执行单轮测试。
```

### 方法 C：全量回归测试

```
所有 10 个 TP 在 2 个版本上各跑一次。
预计总时间：~10 分钟（5 min × 2 版本）
```

## 六、测试注意事项

1. **模型一致性**：跨版本测试使用同一模型（建议 `claude-sonnet-4` 或统一配置）
2. **context 隔离**：每个 TP 独立测试，不共享上下文
3. **输出长度**：记录每次测试的输出字数，用于 FM-1 监控
4. **诚实标注**：subagent 输出截断时如实标注"测试截断"，不算分数
5. **回归检测**：新版本分数不得低于旧版本（棘轮机制）
