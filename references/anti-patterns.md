# Prompt Alchemist — 反模式避坑指南

> 提示词优化的反面教材库：什么不要做、为什么不要做、如何修正。
> 来源：Schulhoff 1565篇论文结论 + 涂津豪意识流陷阱 + 李继刚压缩哲学的反面验证 + 真实案例汇总。

---

## 一、角色设计类反模式

### AP-1：角色堆砌（Role Stuffing）

**症状**：在一个角色里塞入过多属性，希望"全面覆盖"。

❌ **错误示例**：
```
你是一位经验丰富、博学多才、耐心细致、专业严谨、富有创造力的资深专家，
请用通俗易懂、深入浅出、条理清晰的语言为我详细解答以下问题。
```

✅ **修正方向**：用 1-2 个核心属性精准锁定风格。

```xml
<role>量子计算科普专家，擅长用类比解释复杂概念</role>
```

**为什么错**：
- Schulhoff 研究：角色堆砌对准确性任务**效果微弱**（不是无效，是微弱）
- 李继刚：角色即压缩包——属性越多，**信息熵越高**，风格越模糊
- 模型视角：模型会被这些形容词"平均化"，反而失去风格

---

### AP-2：威胁/奖励式提示（Threat/Reward Prompting）

**症状**：用奖惩机制"激励"模型输出。

❌ **错误示例**：
```
答错就扣分！
答得好给你 10 块钱！
如果回答得好我会给你 1000 美元小费！
```

✅ **修正方向**：删除整段奖惩语句。

```xml
<task>请回答以下问题</task>
```

**为什么错**（Schulhoff 研究结论）：
- **威胁**：对输出质量**无显著影响**（p > 0.05）
- **奖励**：效果**不稳定**，但**不如"具体格式要求"有效**
- 唯一例外：**Emotion Prompting**（"这对我的职业生涯很重要"）有微弱正向效果，但属于另一类

---

### AP-3：人格分裂（Persona Splitting）

**症状**：让模型同时扮演多个角色，导致风格冲突。

❌ **错误示例**：
```
请你既是严谨的学术专家，又是幽默的段子手，还要像心理咨询师一样温柔。
```

✅ **修正方向**：选一个主角色，其他作为风格调节器。

```xml
<role>学术专家（主风格）+ 偶尔用类比降低理解门槛（次要调节）</role>
```

## 二、任务定义类反模式

### AP-4：任务混淆（Task Conflation）

**症状**：一个提示词里塞了多个不相关任务。

❌ **错误示例**：
```
帮我写一篇关于秋天的散文，同时把它翻译成英文，再给我一份代码用Python分析文章的情感。
```

✅ **修正方向**：拆为三个独立提示词。

```
提示词 A：写散文
提示词 B：翻译成英文
提示词 C：用Python做情感分析
```

**为什么错**：
- 模型会在不同任务间"切换上下文"，每个任务都做不深
- 输出格式无法统一（一会儿散文，一会儿代码）
- Few-Shot 难以匹配（散文示例 vs 代码示例？）

---

### AP-5：模糊形容词（Vague Adjectives）

**症状**：用"美/专业/详细/好一点"等抽象词描述输出标准。

❌ **错误示例**：
```
帮我写一篇关于秋天的散文，要写得美一点
```

✅ **修正方向**：用具体场景/作家名/感官细节锚定。

```xml
<role>散文作家，笔触介于川端康成的物哀美学与汪曾祺的市井温情之间</role>
<constraints>
- 不出现"金风送爽"等陈词
- 以一个具体场景开篇
- 情感走向：从感官沉浸到时间感知的微痛
</constraints>
```

**为什么错**：
- "美"在向量空间中有成百上千种解读
- 具体场景 + 作家名能在 embedding 中精准命中风格

---

### AP-6：问题三连问（Triple-Question Syndrome）

**症状**：一个提问句里问三件事。

❌ **错误示例**：
```
Python 怎么学？有什么书推荐？学完能找到工作吗？
```

✅ **修正方向**：拆为三个独立提问。

```
1. Python 的学习路径应该怎么规划？
2. 推荐 3 本适合初学者的 Python 书籍
3. 学完 Python 后有哪些就业方向？
```

## 三、格式与结构类反模式

### AP-7：格式重复（Format Repetition）

**症状**：在多处重复相同的格式要求。

❌ **错误示例**：
```
请详细回答。
回答要详细。
要详细。
```

✅ **修正方向**：在 `<format>` 标签中集中定义一次。

```xml
<format>每个维度 200 字以内，关键术语加粗</format>
```

---

### AP-8：元数据缺失（Missing Metadata）

**症状**：不指定目标模型、字数、语言、读者。

❌ **错误示例**：
```
翻译成英文
```

✅ **修正方向**：补充关键元数据。

```xml
<task>将以下中文段落翻译成英文</task>
<format>商务邮件风格（正式但不过于生硬），保留原意，专业术语保持一致</format>
```

最小元数据清单：
- 目标模型（如 `${TARGET_MODEL:claude-sonnet-4.5}`）
- 输出长度（"200 字以内"）
- 风格/场景（"商务邮件"）
- 受众（"给 5 岁小孩讲"）

---

### AP-9：Few-Shot 错位（Mismatched Few-Shot）

**症状**：给的示例与实际任务类型不一致。

❌ **错误示例**（任务是"写产品文案"，示例却是"古诗"）：
```
任务：写一段咖啡机的产品文案
示例：床前明月光，疑是地上霜。
请按此风格写。
```

✅ **修正方向**：示例必须同任务类型、同风格。

```xml
<examples>
风格示例（产品文案）：
"这台咖啡机不是烧水的，是雕刻时间的——30秒，从豆到杯，恰好是一首歌的长度。"
</examples>
```

**为什么错**：
- 模型会"模仿示例的风格 + 误解任务的目标"
- Schulhoff 研究：Few-Shot CoT 在 MMLU 上**始终最优**，但**前提是示例匹配任务**

---

### AP-10：示例顺序错乱（Order Chaos）

**症状**：示例顺序随机，未按"由简到繁"或"由典型到边缘"排列。

❌ **错误示例**（任务：分类）：
```
示例 1：边缘案例（罕见类别）
示例 2：典型案例
示例 3：另一个边缘案例
```

✅ **修正方向**：示例排序原则。

| 排序策略 | 适用场景 |
|---------|---------|
| **由简到繁** | 模型需要"热身"理解任务 |
| **由典型到边缘** | 任务有明确的核心类别 |
| **正反对照** | 任务边界容易混淆（如分类） |
| **时间序列** | 涉及流程/步骤的任务 |

> **Schulhoff 关键发现**：仅调整示例顺序，准确率可从 50% 波动到 90%。

## 四、思考引导类反模式

### AP-11：过度思考引导（Over-Thinking Prompt）

**症状**：简单任务加了复杂的思考指令。

❌ **错误示例**（任务是"翻译 'Hello' 为中文"）：
```
请在回答前进行深入的意识流思考，审视问题的本质、探索多种可能性、
质疑初步结论、回溯验证逻辑链、考虑文化背景、历史渊源、语义演变……
```

✅ **修正方向**：根据任务复杂度匹配思考深度（涂津豪：自适应深度）。

| 任务类型 | 思考指令 |
|---------|---------|
| 简单（翻译、定义） | 无需额外思考引导 |
| 中等（多维分析） | "先分析 X 个维度，再给结论" |
| 复杂（决策、创意） | "在回答前进行意识流思考：审视本质、探索可能性、质疑假设" |

---

### AP-12：思考与回答混淆（Mixing Thinking with Output）

**症状**：思考过程暴露在最终输出里。

❌ **错误示例**：
```
让我想想……首先，这个问题涉及……然后我应该……好的，我的回答是：xxx
```

✅ **修正方向**：用标签隔离思考与输出（涂津豪：内部分离原则）。

```xml
<thinking>
[模型内部思考，不展示给用户]
</thinking>

<response>
[最终输出]
</response>
```

## 五、文化与语言类反模式

### AP-13：中文硬塞英文指令（Forced English Chunks）

**症状**：中文提示词里突然出现未解释的英文术语。

❌ **错误示例**：
```
请用 CoT 的方式 think step by step 然后给我答案
```

✅ **修正方向**（两种选择）：

**选择 A：翻译为中文**
```
请用"逐步思考"的方式（chain of thought）分析问题，最后给出答案
```

**选择 B：保留英文但用括号解释**
```
请按 CoT（chain of thought，思维链）方式，分步骤推理后给出答案
```

---

### AP-14：文化语境错位（Cultural Context Mismatch）

**症状**：示例/类比使用了目标读者不熟悉的文化符号。

❌ **错误示例**（读者是国内小学生）：
```
写一篇关于勇气的文章，风格参考"Eren Yeager 的怒吼"或"高达出击"
```

✅ **修正方向**：锚定读者熟悉的文化符号。

```xml
<role>面向国内小学生的写作教练</role>
<examples>
风格示例：哪吒闹海时，三味真火烧到眼前，他说"我命由我不由天"——这种明知不可为而为之的勇气，就是我们今天要写的。
</examples>
```

## 六、自检速查表

写完提示词后，对照检查：

- [ ] 角色属性 ≤ 3 个？（AP-1）
- [ ] 没有任何奖惩语句？（AP-2）
- [ ] 任务数量 = 1？（AP-4）
- [ ] 模糊形容词 = 0？（AP-5）
- [ ] 格式要求集中在 `<format>` 里？（AP-7）
- [ ] 指定了目标模型/字数/风格？（AP-8）
- [ ] Few-Shot 示例与任务同类型？（AP-9）
- [ ] 示例顺序由简到繁/由典型到边缘？（AP-10）
- [ ] 思考引导匹配任务复杂度？（AP-11）
- [ ] 思考与输出标签分离？（AP-12）
- [ ] 英文术语有中文解释或直接翻译？（AP-13）
- [ ] 文化符号匹配目标读者？（AP-14）

> **12 项全部通过** → 提示词质量合格
> **3 项以上不通过** → 建议重新蒸馏
